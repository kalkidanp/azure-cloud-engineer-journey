# Day 25 — Introduction to CI/CD

## Overview 🎯
Today I learned the fundamentals of CI/CD and how it transforms the way systems are built, tested, and deployed.

This marks the beginning of Week 5, where the focus shifts from building systems to automating and managing them.

---

## What is CI/CD? 🔄

### 🔹 Continuous Integration (CI)
- Code is automatically built and tested
- Ensures changes are safe and do not break the system

### 🔹 Continuous Deployment (CD)
- Code is automatically deployed after passing checks
- Reduces manual work and speeds up delivery

---

## CI/CD Flow 🧠

Code → Build → Test → Deploy → Monitor

---

## Key Benefits 💡
- Faster deployments
- Reduced human error
- Consistent environments
- Scalable workflows

---

## Real-World Thinking 💭

### What happens if deployment fails?
- We can redeploy using versioned templates
- No need for manual reconfiguration

---

### How do we ensure safe updates?
- Test before deployment
- Use automated validation steps

---

### Can CI/CD be used for infrastructure?
- Yes
- Tools like Bicep, Terraform, Azure DevOps, and GitHub enable Infrastructure as Code pipelines

---

## Key Learnings 💡
- CI/CD automates the entire deployment lifecycle
- Infrastructure can be treated like application code
- Automation is essential for modern cloud engineering
