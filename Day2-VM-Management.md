# Day 2 – Azure Virtual Machine Management

## What I Did
- Revisited the VM lab in LabITpro to reinforce understanding
- Watched practical demos on YouTube about starting and stopping Azure virtual machines
- Learned how to configure auto-shutdown for VMs
- Explored VM lifecycle and status changes (Running → Stopped)

## What I Learned 💡
- Virtual machines can be started, stopped, and restarted like physical computers
- When a VM is stopped, compute resources (CPU, RAM) are not used
- Stopping a VM helps reduce cloud costs
- Auto-shutdown is useful for cost optimization and automation
- VM status changes indicate whether the system is actively running or not

## Key Concepts
- **Compute in Azure**: Refers to processing power (CPU, memory) used to run applications
- **VM Lifecycle**:
  - Running → VM is active and consuming resources
  - Stopped → VM is not consuming compute resources
