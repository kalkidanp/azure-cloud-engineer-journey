# Day 30 — Full DevOps Flow (Week 5 Final)

## Overview 🎯
Today I combined everything I learned this week into a complete DevOps workflow.

This includes CI/CD, deployment, monitoring, alerts, and system improvement.

---

## Full System Flow 🔄

1. Developer writes code  
2. Code is pushed to repository  
3. CI/CD pipeline is triggered  
4. Infrastructure / application is deployed  
5. System runs in production  
6. Monitoring tracks performance  
7. Alerts notify issues  
8. Engineers fix the problem  
9. Code is updated and redeployed  

---

## DevOps Loop 🔁

Build → Deploy → Monitor → Alert → Fix → Redeploy → Repeat

---

## Step-by-Step Understanding 🧠

### 🧑‍💻 Code
Developers write application or infrastructure code (Bicep, YAML, etc.)

---

### 🚀 Deployment
Code is deployed using CI/CD pipelines into cloud environments.

---

### 👀 Monitoring
Tools track system health:
- Performance
- Errors
- Traffic

---

### 🔔 Alert
If thresholds are exceeded:
- Notifications are triggered
- Engineers are informed

---

### 🛠️ Fix
Engineers:
- Analyze logs
- Identify root cause
- Apply fixes

---

### 🔁 Redeploy
- Changes are pushed again
- Pipeline redeploys automatically

---

## Key Learnings 💡

- Systems must be automated, observable, and maintainable
- CI/CD connects development to deployment
- Monitoring provides visibility
- Alerts ensure fast response
- DevOps is a continuous loop, not a one-time action
