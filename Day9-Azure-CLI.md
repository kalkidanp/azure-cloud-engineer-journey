# Day 9 – Azure CLI (Command Line Interface)

## What I Did
- Studied Azure CLI concepts and its cross-platform nature
- Explored how Azure CLI interacts with Azure services
- Learned how CLI can be used for automation and scripting
- Installed Azure CLI locally and explored command structure
- Practiced understanding commands without memorizing them
- Watched demonstrations on creating a Resource Group using CLI
- Learned how to use interactive mode and change output formats

---

## Key Takeaways 💡
- Azure CLI is a cross-platform tool (works on Windows, Linux, macOS)
- It allows managing Azure resources using commands instead of the portal
- CLI is widely used for automation and scripting in real-world scenarios
- Understanding command structure is more important than memorizing commands
- Developers and engineers rely on CLI for speed and efficiency

---

## Key Concepts
- **Azure CLI**: Command-line tool to manage Azure resources
- **Cross-Platform**: Can run on multiple operating systems
- **Interactive Mode**: Helps guide users through commands
- **Output Formats**: JSON, table, and others for different use cases
- **Automation**: Using scripts to perform repetitive tasks

---

## Example Commands 💻

```bash
az login
az group create --name myRG --location eastus
az group list --output table
👉 These commands:
- Authenticate user
- Create a Resource Group
- Display resources in readable format
## Deeper Understanding 🔍
- Azure CLI enables automation, making cloud operations faster and scalable
- It is commonly used by developers in CI/CD pipelines
- CLI integrates well with scripting tools for Infrastructure as Code
