# Day 5 – Azure Networking (VNet & Connectivity)

## What I Did
- Completed a lab on creating and configuring an Azure Virtual Network (VNet) using LabITpro
- Configured address space and subnets
- Identified unclear networking and security concepts and researched them
- Explored how virtual machines connect to VNets and access the internet
- Watched demonstrations explaining VM and VNet connectivity and networking rules

## What I Learned 💡
- A Virtual Network (VNet) is a private network in Azure where resources are deployed
- Subnets divide a VNet into smaller segments for better organization and security
- Virtual machines are deployed inside a VNet and communicate using private IP addresses
- Public IP addresses allow resources to be accessed from the internet
- Subnets help isolate resources and control traffic flow

## Key Concepts
- **VNet**: Private cloud network
- **Subnet**: Segmented portion of a VNet
- **Public IP**: Enables internet access
- **Private IP**: Used for internal communication
- **IP Range (CIDR)**: Defines the size and structure of the network

## Additional Concepts Explored 🔍
- **Azure Bastion**: Secure way to connect to VMs without exposing them to the internet
- **Azure Firewall**: Controls and filters network traffic
- **DDoS Protection**: Protects against large-scale network attacks
- **Private Endpoint**: Secure connection to services within a private network
- **VNet Encryption**: Protects data within the network
- **Tags**: Used for organizing and managing resources

## Deeper Understanding
- Virtual machines are connected to VNets through network interfaces
- Internet access for VMs is controlled through public IPs and network configurations
- Subnets improve security by separating resources and limiting access

## Challenges ⚠️
- Networking and security concepts were initially complex
- Needed additional research and videos to understand connectivity and configurations

## Notes / Reflection 🚀
- Networking is the backbone that connects all cloud resources
- Understanding how VMs connect to VNets and the internet clarified many cloud concepts
- Feeling more confident in understanding how real cloud environments are structured
