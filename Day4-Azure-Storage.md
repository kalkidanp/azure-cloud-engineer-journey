# Day 4 – Azure Storage Account & Data Management

## What I Did
- Completed a lab on creating and configuring an Azure Storage Account using LabITpro
- Explored storage account settings such as region, performance, and redundancy
- Identified unclear configurations and researched them for deeper understanding
- Studied how storage accounts and virtual machines are connected
- Watched demonstrations on creating containers and uploading files in Azure Blob Storage

## What I Learned 💡
- A Storage Account is the foundation for storing data in Azure
- Blob storage is used to store unstructured data like images, videos, and documents
- Disk storage is attached to virtual machines and acts as their internal storage
- Redundancy options (LRS, GRS) help protect data and ensure high availability
- Data in Azure is stored in specific regions, which affects performance and latency
- Not all data should be stored on a VM because it is not scalable or cost-efficient

## Key Concepts
- **Storage Account**: Central resource that holds Azure storage services
- **Blob Storage**: Object storage for files and unstructured data
- **Disk Storage**: Storage attached to virtual machines
- **Container**: Logical grouping for storing blobs (files)
- **Redundancy**: Replication of data to prevent loss

## Deeper Understanding 🔍
- Virtual machines rely on disk storage for system operations, but scalable data storage is handled separately using services like Blob storage
- Separating compute (VM) and storage improves flexibility, performance, and cost efficiency
- Storage accounts allow centralized and secure data management

- The lab did not include container creation or file upload, requiring additional research
