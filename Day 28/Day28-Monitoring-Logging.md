# Day 28 — Monitoring & Logging

## Overview 🎯
Today I learned how to observe and understand system behavior using monitoring and logging.

This is essential because building a system is not enough — we must also know what is happening inside it.

---

## Monitoring vs Logging 🧠

### 🔹 Monitoring 📊
Monitoring is the process of tracking system performance and health in real time.

Examples:
- CPU usage
- Response time
- Number of requests

💡 Analogy:
Monitoring is like a **security guard** watching the system live.

---

### 🔹 Logging 📄
Logging records events and activities that happen inside the system.

Examples:
- Errors
- User actions
- System events

💡 Analogy:
Logging is like a **diary** that records everything that happened.

---

## Key Difference ⚖️

- Monitoring → tells *what is happening now*
- Logging → explains *why it happened*

---

## Why Both Are Important 💡

- Monitoring helps detect issues quickly  
- Logging helps investigate and fix problems  
- Together, they provide full system visibility  

---

## Azure Tool ☁️

- Azure Monitor → collects metrics and logs

---

## Real-World Thinking 💭

If a system becomes slow:
- Monitoring shows performance drop  
- Logs help identify the root cause  

---

## Key Learnings 💡

- Systems must be observable
- Monitoring gives visibility
- Logging gives explanation
- Both are required for troubleshooting
